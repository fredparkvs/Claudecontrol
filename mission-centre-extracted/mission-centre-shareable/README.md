# Mission Centre

A PowerShell TUI launcher for AI projects. Auto-discovers projects across a folder tree, tracks them in a local registry, shows recent Claude Code session history with auto-generated titles, and launches sessions via [Jig](https://github.com/BarendL/jig) profiles or the `claude` CLI directly.

## What it looks like

```
╔════════════════════════════════════════════╗
║               MISSION CENTRE               ║
╚════════════════════════════════════════════╝

 [1]  My Web App [corporate]
      Firebase + React dashboard

 [2]  Side Project [vibe]
      Weekend experiment

──────────────────────────────────────────────

  Recent meta missions:
   [M1] refactor auth flow  (2d ago)

  [M]   New Meta Mission - cross-project planning
  [M#]  Resume a meta mission listed above

  Pick a project number, [M] Meta Mission, [S] Scan, or [Q] Quit
```

Inside a project:

```
  Recent missions:
   [R1] firebase production migration  (0m ago)
   [R2] fix login redirect loop  (3d ago)

  [L]  Launch                - jig run corporate (project default)
  [J]  Launch with Jig       - pick a different profile
  [R#] Resume mission        - resume a session listed above
  [E]  Open in Explorer
  [B]  Back
```

Session titles like `firebase production migration` are auto-generated by Claude Code — no manual bookkeeping needed.

## The tool stack

Mission Centre is the front door to a three-tool workflow:

### 1. Claude Code (`claude`)
The AI coding CLI from Anthropic. Each session is a conversation with Claude in your project directory. Sessions are stored locally and resumable.
- Install: https://claude.ai/code
- Docs: https://docs.anthropic.com/claude-code

### 2. Jig
A Go CLI that manages **session profiles** for Claude Code — each profile bundles a set of skills, MCP servers, and permissions so you can launch the right Claude configuration for the job (e.g. `corporate` vs `vibe` vs `build`).
- Repo: https://github.com/jdforsythe/jig *(by [@jdforsythe](https://github.com/jdforsythe))*
- Install: download the binary and add to PATH, or `go install` from source
- Usage: `jig` — interactive picker; `jig run <profile>` — launch a named profile

Without Jig, Mission Centre still works — use **[J]** to open a plain `jig` picker, or **[R#]** to resume sessions directly via `claude --resume`.

### 3. Forge *(optional)*
An agent-assembly system that uses Claude Code skills to plan and build AI agent teams for complex projects. Mission Centre's **[M] Meta Mission** launches a Forge planning session from your scan root.
- Repo: https://github.com/jdforsythe/forge *(by [@jdforsythe](https://github.com/jdforsythe))*
- Forge lives in your Claude Code skills/plugins directory
- Without Forge, the Meta Mission option just opens a plain Claude Code session

## Prerequisites

- **Windows 10/11** with PowerShell 5.1+ (built in) or PowerShell 7+
- **Claude Code** (`claude`) — install from https://claude.ai/code
- **Jig** (`jig`) in PATH — optional but recommended for profile-based launching

---

## Setup

1. Clone this repo somewhere permanent:
   ```
   git clone https://github.com/BarendL/mission-centre C:\Tools\mission-centre
   ```

2. Run via the batch file:
   ```
   C:\Tools\mission-centre\launch.bat
   ```
   Or right-click `launch.bat` → Run. The window stays open on errors so you can see what went wrong.

3. **First run:** you will be prompted for your **scan root** — the top-level folder Mission Centre searches for projects (e.g. `C:\AI` or `C:\Dev`). Saved to `config.json` (gitignored).

4. Press **[S]** to scan. For each discovered project you will be prompted for a name, description, and Jig profile. Press Enter to accept defaults.

## Jig profiles

When adding a project, assign it a profile name. This is passed to `jig run <profile>` when you press **[L]**. Suggested names:

| Profile | Intended use |
|---------|-------------|
| `standard` | General-purpose session (good default) |
| `corporate` | Client / work projects |
| `vibe` | Personal / experimental projects |
| `build` | Greenfield or new feature work |
| `maintain` | Bug fixes and maintenance |
| `review` | Code review sessions |

Profile names are free-form — they just need to match a profile defined in Jig. If Jig is not installed, profile names are ignored.

## Files

| File | Purpose |
|------|---------|
| `Mission-Centre.ps1` | Main script |
| `launch.bat` | Double-click launcher (pauses on error) |
| `config.json` | Your scan root — created on first run, **gitignored** |
| `projects.json` | Your project registry — built by scanning, **gitignored** |
| `config.example.json` | Template for config.json |
| `projects.example.json` | Template for projects.json |

`config.json` and `projects.json` are gitignored because they contain your personal paths and project names.
